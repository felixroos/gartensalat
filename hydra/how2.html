Continuing how hydra works, let's see how to turn a hydra style patch into GLSL code. With the help of graph compilation, we can treat each function call as a node.

In the last episode, we've seen that hydra has generated code like this: 

color(mult(osc(rotate(st, 0., 0.1), 100., 0.01, 1.4), osc(modulate(st, osc(rotate(st, 0., -0.1), 10., 0.1, 0.), 1.), 10., 0.1, 0.), 1.), 2.83, 0.91, 0.39, 1.)

Instead of one nested call expression, we can write the same calculation in multiple steps like this: 

vec2 v6 = rotate(st, 0., -0.1);
vec4 v5 = osc(v6, 10., 0.1, 0.);
vec2 v4 = modulate(st, v5, 1.);
vec4 v3 = osc(v4, 10., 0.1, 0.);
vec2 v2 = rotate(st, 0., 0.1);
vec4 v1 = osc(v2, 100., 0.01, 1.4);
vec4 v0 = mult(v1, v3, 1.);
color(v0, 2.83, 0.91, 0.39, 1.);
see how i did it

Here, I've manually de-nested the expression by creating a variable for each step. This would be the target output of our graph compiler. Let's take the inverse route and manually morph that into our input language. We will follow this one rule: a.b(c) = b(a,c), for example rotate(st, 0, -0.1) = st.rotate(0, -0.1). By stupidly applying this rule throughout, we end up with this: 

st.rotate(0, 0.1)
.osc(100, 0.01, 1.4)
.mult(
  st.modulate(
    st.rotate(0, -0.1)
    .osc(10, 0.1, 0),
    1
    )
  .osc(10, 0.1, 0), 
  1
)
.color(2.83, 0.91, 0.39, 1)
see how i did it

This exact code (+ .out() at the end) can now be used as input to our graph compiler:
